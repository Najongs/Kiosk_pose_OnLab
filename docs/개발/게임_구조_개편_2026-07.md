# 게임 선택 구조 개편 + 미니게임 3종 추가 (2026-07)

> 단일 스트레칭 앱 → **다중 게임 키오스크**로 재편한 작업 기록.
> 배경: [콘텐츠/아이디어_적용성_분석.md](../콘텐츠/아이디어_적용성_분석.md)의 1등급 후보 구현.

## 무엇이 바뀌었나

### 1. 홈 화면 = 게임 선택
- 우측 패널이 **게임 카드 5장**(스트레칭 코스 / 2인 대결 / 반응속도 / 높이뛰기 / 팔굽혀펴기)으로 교체. 스트레칭 카드 → 기존 코스 선택이 하위 페이지로 열림(← 뒤로가기).
- 리더보드가 **게임별 탭**으로 분리. 기록에 `game` 키 추가 (`core/leaderboard.py`), 키 없는 과거 기록은 stretch 로 간주 — 하위 호환.
- 좌측 "빠른 시작"은 기본 스트레칭 세트 즉시 시작(기존 동작 유지). ⚔ 2인 대결 버튼은 카드로 이동.

### 2. 신규 미니게임 3종
| 게임 | 로직 | 판정 방식 | 비고 |
|---|---|---|---|
| 반응속도 | `core/games/reaction.py` | 신호(화면 플래시) 후 손목이 어깨 위로 올라올 때까지 ms | 부정 출발 감지·감점. 카메라·추론 지연 포함이라 상대 경쟁용 |
| 높이뛰기 | `core/games/jump.py` | 2초 정지 캘리브레이션 → 머리(코) 키포인트 상승량을 cm 근사(몸통≈50cm) | 목표선+공 렌더. "약 Ncm" 표기 |
| 팔굽혀펴기 | `core/games/pushup.py` | 팔꿈치 각도 up(>150°)→down(<95°)→up 전이로 rep 카운트 | 어깨-엉덩이-발목 정렬 <160° 시 자세 경고, 바른 자세 비율이 점수 반영 |

공통 설계: `core/session.py` 패턴 미러 — 상태 Enum + 렌더 계약 dataclass +
`update(primary, now)`, **시간·난수(rng) 주입**으로 카메라 없이 결정적 테스트 가능.
설정 키(라운드 수·제한시간·각도 임계값 등)는 `core/appconfig.py` DEFAULT_APP_CONFIG 참고.

### 3. 아키텍처 리팩터링
- **`ui/game_view.py`** — `BaseGameView`: SessionView/VersusView 에 중복돼 있던 워커
  스레드 수명주기(카메라 열기·추론 스레드·표시·실패/종료 처리)를 추출. 기존 두 뷰도
  이 위로 이전(공개 `begin` 시그니처는 불변 — 검증 스크립트 무수정 통과).
  `MiniGameView`: 1인 게임 공통(추정기+추적기+게임 조립, 사운드 큐, 리더보드 기록).
- **`ui/game_registry.py`** — 홈 카드·화면 전환·리더보드 탭의 단일 출처(`REGISTRY`).
  `MainWindow`는 하드코딩된 3-widget 스택 대신 선택 시 뷰를 지연 생성.
- **`ui/hud.py`** — 등급 배지·메시지 필·진행 도트·컨페티·카운트다운 링을
  `renderer.py`에서 추출(추출 전후 렌더 출력 픽셀 해시 동일 확인).
- **`ui/game_renderers.py`** — 미니게임 3종 화면 합성(기존 네온 룩 유지).

### 4. 새 게임 추가 절차 (이후 확장용)
1. `core/games/<게임>.py` 상태머신 → 2. `compose_<게임>` 렌더러 → 3. `MiniGameView`
서브클래스 → 4. `REGISTRY` 한 줄 → 5. `test_games.py`/`verify_games.py` 케이스.
상세는 [CLAUDE.md](../../CLAUDE.md)와 [아키텍처_개요.md](아키텍처_개요.md) 참고.

### 5. 게임 룩 연출 강화 (후속 폴리시)
- **`ui/hud.py` 연출 프리미티브**: 비네트(가장자리 어둡게, 해상도별 마스크 캐시),
  코너 브래킷(호흡 펄스), 상단바 흐르는 액센트 라인, 결과 화면 회전 광선(burst rays),
  신호 확산 링, 카운트다운 링 업그레이드(시계 눈금 + 초 경계 플래시 + 발광 끝점)
- **게이지**(`core/drawing.gauge_bar`): 상단 하이라이트 + 발광 캡 + 흐르는 샤인(anim_t)
- 게임별 디테일: 반응속도(신호 팝 + 확산 링, 대기 중 화면 딤 + 숨쉬는 점),
  높이뛰기(줄에 매달려 까딱이는 공, 점프 중 스피드 라인 + 실시간 cm),
  팔굽혀펴기(down 자세에서 카운터 골드 점화, 마지막 5초 붉은 깜빡임, 굽힘 게이지에
  인정선 눈금), 스트레칭(합격 시 브래킷 골드, 콤보 펄스, 카운트다운 숫자 팝)
- **홈 화면**: 타이틀·시작 버튼·게임 카드에 액센트색 네온 글로우
  (QGraphicsDropShadowEffect), 이모지 원형 배지, PLAY 화살표
- **주의**: PIL 렌더 텍스트에 이모지 사용 금지 — 한글 폰트(맑은고딕/Noto CJK)에
  글리프가 없어 □ 로 깨진다 (⏰🔥 로 실제 발생, 텍스트로 교체). Qt 위젯 쪽 이모지는 무관.

### 7. 플레이 중 그래픽 강화 (2026-07)
- **스포트라이트**(`ui/hud.spotlight`): 플레이 중 사람 주변만 밝게 — 48px 그리드
  마스크 캐시로 프레임당 ~1.3ms. 비활성 시 비네트(`stage_light` 로 자동 전환)
- **모션 트레일**(`ui/hud.TrailTracker`): 손목·머리 잔상 — 렌더 클로저가 상태 소유
- **이벤트 팝업+스파크**(`ui/hud.draw_popups` + `MiniGameView._fx_events` 훅):
  팔굽혀펴기 +1(나쁜 자세 rep 은 붉은기), 점프 착지 +Ncm
- **"시작!" 스플래시**(`ui/hud.splash_text`): 스트레칭 채점 진입 순간
- **게임별 스켈레톤 테마 색**: 홈 카드 액센트와 일치 (점프 하늘색,
  팔굽혀펴기 주황→무너지면 빨강, 반응속도 민트→신호 골드)
- 성능 기준: 합성 전체 ~15ms/frame(720p) — 30fps 예산의 절반 이하 유지.
  연출 추가 시 `spotlight` 벤치처럼 ms 단위로 재고 넣을 것

## 검증 (전부 헤드리스, 카메라 불필요)

```bash
python tools/test_games.py     # 미니게임 상태머신: 상태 전이·부정출발·rep·자세경고
python tools/verify_games.py   # 미니게임 뷰 오프스크린: DONE 도달 + 게임별 리더보드 기록
python tools/verify_ui.py      # 기존 스트레칭 회귀 (리팩터링 후 무수정 통과)
python tools/verify_versus.py  # 기존 대결 회귀
```

추가로 MainWindow 5개 게임 진입/Esc 복귀 스모크, HUD 추출 전후 픽셀 해시 비교 수행.

## 이번에 발견해 고친 것 (실험 기록 성격)

- **falsy 0.0 버그**: `now - (self._calib_start or now)` — 시작 시각이 0.0이면
  falsy 로 빠져 경과시간이 항상 0. 게임 로직 전반을 `is not None` 비교로 통일.
  기존 `session.py`의 `(deadline or now)` 패턴은 deadline이 0이 될 일이 없어 무사했음.
- **DONE 기록 경합**: 워커 스레드가 게임을 DONE 으로 전이시킨 직후 검증 루프가
  종료→`stop()`으로 시그널을 끊으면, 큐에 있던 DONE 스냅샷이 처리되지 않아
  리더보드 기록이 누락될 수 있음(간헐 실패). 검증 스크립트는 종료 후 몇 프레임
  플러시하도록 수정. 실제 앱은 사용자가 결과 화면을 보고 나가므로 영향 없음.
- **자세 경고는 좌우 평균이 아니라 min**: 한쪽만 처져도 경고해야 하므로
  `body_line_angle`은 더 굽은 쪽을 반환.

### 커밋 전 다중 에이전트 코드 리뷰(high)에서 나와 고친 것
- **실패에도 성공음**: 반응속도 REST 상태를 통째로 성공음 대상으로 지정했었는데,
  부정 출발·시간 초과도 REST 로 진입 → 정상 반응일 때만 성공음이 나도록 뷰에서
  직접 판정 (`timed_out` 플래그를 상태에 추가).
- **게이지-카운트 기준 불일치**: 팔굽혀펴기 굽힘 게이지가 150°/95°를 하드코딩 —
  설정(`pushupUpAngle/DownAngle`)을 바꾸면 시각 피드백과 실제 카운트 기준이 어긋남.
  상태 스냅샷에 각도를 실어 렌더러가 같은 눈금을 쓰도록 수정.
- **지연 뷰 생성이 오류 가드 밖**: 게임 카드 클릭 시 `make_view()`(지연 import)가
  실패하면 홈에 "시작 실패" 안내 없이 예외 전파 → try 안으로 이동.
- **falsy 0.0 재발**: `self._peak or hy` — 점프 정점이 화면 최상단(y=0)이면 기록이
  버려짐. `is not None` 비교로 수정 (캘리브레이션 버그와 같은 계열).
