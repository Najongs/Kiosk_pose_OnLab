# ONLAB × 우송대 협력과제 — 회의 안건 및 질문 리스트

> 작성 관점: **ONLAB(주관사)** → 우송대(개발 협력) 대상 회의 자료
> 목적: 키오스크 기반 자세추정(스켈레톤) 인터랙션 과제의 범위·시스템·역할·일정 확정
> 표기: `▢ 기재/확정` = ONLAB 확정·기재, `▶ 제안` = ONLAB 제안, `▷ 우송대 협의` = 우송대와 합의/요청, `※` = 참고
>
> ※ 추론 백본은 **MediaPipe 권장**(프로토타입 검증 완료, 라이선스·CPU 실시간·개발속도 우위).
>   OpenPose는 대안으로 2.2에서 비교. 프로토타입: MediaPipe + Python(PySide6) / 웹(TS+WASM).

## 2. 과제 소개

### 2.1 사용 시나리오 / 어플리케이션 / 콘텐츠

**핵심 정의(합의 필요):** "어떤 사용자가 / 어떤 환경에서 / 어떤 경험을 / 어떤 흐름으로 얻는가"

- `▢ 기재/확정` 주 사용자층 (연령·목적), 설치 환경 (전시장·행사·매장·체육시설), 1회 이용시간
- `▢ 기재/확정` 동시 이용 인원 (1인 / 다인), 대기·전환 흐름
- `▢ 기재/확정` 콘텐츠 성격 우선순위: 교육형 / 오락형 / 행사·체험 중 비중

**▶ 제안 — 콘텐츠 4종 + 표준 사용자 시나리오**

공통 흐름: `대기(사람 감지) → 이름/모드 선택 → 카운트다운 → 수행/평가 → 결과·점수 → 리더보드/공유`

| 콘텐츠 | 유형 | 사용자 시나리오(예) | 판정 방식 |
|--------|------|--------------------|-----------|
| **Body Quiz** | 오락/교육 | "오른손 들어, 왼손 내려" 지시 → 자세로 정답 | 관절 위치/각도 규칙 매칭 |
| **Motion Game** | 오락 | 화면의 풍선을 손끝으로 터치해 터뜨리기 | 손목/손끝 좌표 히트박스 |
| **운동 교육/코칭** | 교육 | 스쿼트·스트레칭 자세 분석 → 정확도·교정 가이드 | **관절 각도 기반 채점**(현 프로토타입) |
| **Dance / 유연성 배틀** | 오락 | 2인 동시 수행 → 유사도/점수 비교, 승자 | Similarity(자세 벡터) / 각도 점수 + **2인 대결**(구현됨) |

**※ 현재 구현 완료(프로토타입)로 즉시 시연 가능한 것**
- 유연성/스트레칭 자세 채점(앞으로 굽히기·측면 굽히기·만세·T포즈·한발 밸런스)
- 유지시간 판정, 정확도 게이지, **유연성 리포트(관절 각도·등급·좌우 비대칭)**
- **난이도/테마 코스**, **리더보드**, 음성/효과음 안내, 목표 자세 가이드, 관리자 화면
- **2인 실시간 대결**(좌/우 플레이어, 분할 화면, 승자 판정)

`▶ 제안` 위 4종을 "공통 인터랙션 엔진 + 콘텐츠 플러그인" 구조로 개발 → 콘텐츠 추가가 쉬움.

### 2.2 시스템 구성

| 항목 | 사양 (확정 필요) | ▶ 제안/참고 |
|------|------------------|-------------|
| **Camera** | `▢` 해상도 / FPS / 시야각(FOV) | 전신 확보 위해 광각(≥90°), 1080p·30fps 이상 권장 |
| **Compute** | `▢` Jetson Orin NX (모듈/RAM, 8GB vs 16GB) | MediaPipe는 CPU만으로도 실시간 → 저사양 여유. Jetson 시 GPU 델리게이트 |
| **Display** | `▢` 크기 / 해상도 / **터치 여부** | 키오스크는 세로형 다수 → 레이아웃 영향 |
| **Speaker** | `▢` 사용 여부 | 음성 안내(TTS)·효과음 설계에 필요 |
| **OS** | `▢` Ubuntu 버전 (Jetson=JetPack L4T) | JetPack 버전이 CUDA/드라이버 결정 |
| **추론 백본** | `▶` **MediaPipe Pose(권장)** / OpenPose(대안) | 아래 비교표 참고 |
| **개발 언어** | `▢` C++ / Python 경계 | Python 단일로 개발속도↑ (현 프로토타입). 성능 병목 시 C++ 부분 최적화 |
| **개발 프레임워크** | `▢` PySide6/Qt(권장) / Unity | 프로토타입=PySide6(데스크톱)+웹. 3D/이펙트 강하면 Unity |

**추론 백본 비교 (권장: MediaPipe)**

| | **MediaPipe Pose** (권장) | OpenPose (대안) |
|---|---|---|
| 실시간성 | CPU만으로 실시간(~30fps), Jetson 여유 | GPU 필수, Jetson서 최적화 필요 |
| 라이선스 | Apache-2.0 (상업 이용 자유) | 비상업/상업 별도 라이선스 이슈 |
| 3D 좌표 | 제공(world landmarks) → 각도 채점 유리 | 2D 중심 |
| 다중 인원 | numPoses 설정(2인 대결 구현됨) | 다인 강점(혼잡 환경) |
| 개발 성숙도 | 프로토타입 검증 완료 | 빌드·의존성 복잡 |

**핵심 확인 질문**
- `▷ 우송대 협의` 추론 백본 **MediaPipe 권장** 수용 여부 (OpenPose 필요 사유가 있으면 비교 재검토)
- `▢` 다중 인원(2인 대결) 요구 수준 / 혼잡 환경 여부 → 백본·모델 선택에 영향
- `▢` 최종 산출물 형태: 단일 키오스크 앱 / 웹 배포 / 둘 다 (프로토타입은 양쪽 보유)

### 2.3 AI 성능평가

`▶ 제안 — 평가 지표(합의 필요)`
- **키포인트 정확도**: PCK / MPJPE 등 (기준 데이터셋 또는 자체 촬영셋)
- **실시간성**: Jetson에서 FPS, 프레임 지연(ms) — 목표 예: ≥15fps, 지연 ≤100ms
- **강건성**: 조명·거리·복장·부분 가림(occlusion)·다중 인원 상황
- **인터랙션 정확도**: 제스처/자세 판정 정확도, 오탐률
- **채점 타당성**: 전문가 라벨 대비 점수 상관, 재현성(같은 자세 반복 시 점수 편차)
- `▢` 합격 기준(수용 기준)과 평가용 데이터 확보 주체

## 3. R&R 협의

### 3.1 우송대(개발팀)에 대한 기대 사항

전체 파이프라인 개발/최적화:

```
Camera → Frame Acquisition → Pose Estimation(MediaPipe/OpenPose) → Skeleton
      → Gesture/Pose Recognition → Interaction Engine → Application → Display
```

- 개발 환경 구축 (Jetson JetPack, 포즈추정 런타임, 카메라 연동, 프레임워크 셋업)
- 인터랙션 엔진(자세·제스처 판정·채점) 및 콘텐츠 구현
- 성능 최적화(TensorRT 등) 및 안정화
- `▢` 기대 산출물·범위의 명확한 경계 (어디까지가 학생 담당인지)

### 3.2 우송대 학생에 대한 기대 사항

- `▷ 우송대 협의` 참여 인원/전공/수준, 투입 기간(학기·주당 시간)
- `▷ 우송대 협의` 담당 역할 분담 (비전/AI, 앱/UI, 통합/QA)
- `▶ 제안` 정기 산출물 제출(주/격주 리포트), 형상관리(Git) 사용

### 3.3 우송대 학생에 대한 지원 사항

- `▢` 하드웨어 대여 (Jetson, 카메라, 키오스크/디스플레이)
- `▢` 개발 환경/라이선스, 데이터셋, 멘토링·현업 리뷰
- `▢` 인건비/장학금/인턴십, 산학 협약·지식재산(IP) 귀속

## 부록. 회의에서 반드시 확정할 5가지
1. 추론 백본 확정 (**MediaPipe 권장** vs OpenPose) + 다중인원 요구
2. 하드웨어 사양 (카메라·Jetson·디스플레이/터치·스피커)

4. 산출물·역할 경계 (엔진/콘텐츠/통합)와 학생 지원(HW·IP·보상)
