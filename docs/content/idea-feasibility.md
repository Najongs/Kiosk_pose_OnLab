# 행사용 키오스크 콘텐츠 아이디어 — 적용성 분석

> 2026-07 브레인스토밍 아이디어를 **현재 코드베이스로 얼마나 쉽게 구현 가능한지** 기준으로 분류.
> 판단 기준은 아래 "현재 보유 자산" — 재사용 가능한 모듈이 많을수록 등급이 높다.

## 현재 보유 자산 (재사용 가능한 것)

| 자산 | 위치 | 내용 |
|---|---|---|
| 포즈 추정 | `core/mediapipe_estimator.py` | 33키포인트 + 3D world landmarks, 다인 검출(`num_poses`), LIVE_STREAM 비동기 |
| 주 피사체 추적 | `core/tracker.py` | 군중 속 1명 고정 추적 (IoU + grace) |
| 각도 채점 | `core/scorer.py`, `core/geometry.py` | 관절각/기울기 기반 — 체격·위치 무관, JSON으로 자세 정의 |
| 유지 판정 | `core/hold.py` | 임계값 n초 유지 + 흔들림 유예 |
| 세션 상태머신 | `core/session.py` | 대기→카운트다운→채점→결과→다음, 콤보, 이탈 처리 |
| 2인 대결 | `core/versus.py`, `ui/versus_view.py` | 화면 좌/우 분할 배정, 중복검출 제거, 승패 판정 |
| 유연성 리포트 | `core/report.py` | 관절별 ROM 등급 + 좌우 비대칭 경고 |
| HUD/연출 | `ui/renderer.py` | 게이지, 유지 바, 콤보, 등급 배지, 컨페티, 네온 스켈레톤 |
| 운영 인프라 | 리더보드, 관리자(PIN)·코스 편집, 어트랙트 모드, 사운드/BGM, PyInstaller 배포 |
| 자세 자동 생성 | `tools/import_poses.py` | 참고 이미지 → 채점 가능한 자세 JSON 자동 생성 |

**핵심 판단 축**: ① 포즈 파이프라인 재사용 여부 ② 신규 하드웨어 필요 여부 ③ 외부 서비스(온라인) 의존 여부 ④ 단일 정면 카메라 한계(깊이 방향 측정 부정확)에 걸리는지.

## 1등급 상세 — 구현 방향

### 1. 저스트 댄스 (K팝 동작 따라하기)
현재 앱과의 차이는 "정지 자세 유지"가 "시간에 따라 변하는 자세 시퀀스"가 되는 것뿐이다.

- **재사용**: `scorer.py`(프레임별 채점 그대로), `renderer.py`(HUD/콤보/등급), 리더보드, 코스 개념(곡 = 코스)
- **신규**:
  - 안무 기준 데이터: 안무 영상을 `tools/import_poses.py` 방식으로 프레임 샘플링 → 타임스탬프별 자세 JSON 시퀀스 생성 (신규 툴 `tools/import_choreo.py`)
  - `session.py` 확장 또는 신규 `dance_session.py`: 유지 판정 대신 "해당 시점 ±윈도 내 최고 일치율" 채점
  - 가이드 표시: 기존 베이크 스프라이트 파이프라인(`tools/bake_character.py`)이 시퀀스 재생을 이미 지원
- **주의**: 음원 저작권 — 행사용이면 저작권 프리 음원 또는 라이선스 확인 필요
- **예상 규모**: 중 (신규 모듈 1–2개 + 안무 임포트 툴)

### 2. 리듬게임 (점프 / 손 흔들기)
- **재사용**: 추정기, 추적기, HUD, 사운드, 리더보드
- **신규**: 키포인트 이벤트 감지기 — 점프는 엉덩이/발목 y좌표의 급상승, 손 흔들기는 손목 x좌표의 주기적 진동. 판정 타이밍(노트)과 매칭하는 단순 판정 루프
- **주의**: 카메라 지연(약 1프레임 30ms + 추론 지연)만큼 판정 윈도를 넉넉히
- **예상 규모**: 소–중

### 3. 반응속도 / 순발력 테스트
- **재사용**: 세션 상태머신 골격, HUD, 리더보드, 어트랙트
- **신규**: "화면 신호 → 지정 동작(손 들기/점프) 감지까지 시간 측정" 루프 하나. 자세 정의 JSON에 "왼손 들기" 같은 간단 포즈 추가만으로 감지 가능
- **버튼 하드웨어 없이 시작 가능** — 반응 좋으면 그때 버튼 도입 검토
- **예상 규모**: 소 (가장 빨리 만들 수 있는 신규 콘텐츠)

### 4. 팔굽혀펴기 / 근력 자세검사
- **재사용**: `geometry.py`(팔꿈치 각도), `hold.py`의 상태 전이 패턴, `report.py`(자세 품질 리포트 — 허리 처짐을 어깨-엉덩이-발목 정렬각으로 감지)
- **신규**: rep 카운터 — "내려감(팔꿈치 <90°) → 올라옴(>160°)" 상태 전이 카운팅
- **주의**: 팔굽혀펴기는 측면 카메라가 정확 — 키오스크 배치 시 사용자를 옆으로 서게 안내 필요
- **예상 규모**: 소–중. 유연성 검사와 묶어 "신체검사 코스"로 구성 가능

### 5. 같이 하는 스트레칭 (다인 동시)
- **재사용**: `num_poses` 다인 추정은 이미 동작, `versus.py`의 화면분할·플레이어 배정 로직, 기존 스트레칭 자세/코스 전부
- **신규**: 대결이 아닌 "협동 모드" UI 변형 (전원 통과 시 보너스 등)
- **예상 규모**: 소 — 기존 versus의 스킨 변경에 가까움

### 6. 높이뛰기 (공 헤딩) 측정
- **재사용**: 추정기, HUD, 리더보드
- **신규**: 기준선(서 있을 때 머리 y) 대비 점프 정점의 머리 키포인트 상승량 측정. 화면에 가상의 공/목표선 렌더링(`renderer.py`에 오버레이 추가)
- **주의**: 픽셀→cm 환산은 사용자 신장 입력 또는 world landmarks로 보정
- **예상 규모**: 소

## 2등급 비고

- **터치 UI 계열** (운세·성향테스트·만족도조사·투표·시력테스트·틀린그림찾기): 구현 난이도는 낮지만 포즈 파이프라인과 무관한 별도 화면 개발. 행사 운영 관점에서 **만족도 조사/투표**는 실용성이 높아 우선 검토 가치 있음 (PySide6 화면 + JSON 저장이면 충분).
- **달리기 분석**: 제자리 뛰기 케이던스·무릎 높이는 정확히 측정 가능. "속도 km/h"는 추정 공식이라 재미 요소로만.
- **야구 폼**: 손목 이동 속도는 계산 가능하나 투구는 카메라 방향(깊이)으로 움직여 단일 정면 카메라 한계에 정면으로 걸림. 측면 배치 시 개선.
- **얼굴 넣기**: MediaPipe Face Detection/Mesh 추가로 가능. 포즈 자산 재사용은 없지만 MediaPipe 스택은 공유. "박물관 명화 얼굴 합성" 같은 단순 크롭-합성이면 난이도 낮음.
- **그림 미디어아트**: 종이 그림 촬영 → 배경 제거(색 임계 또는 세그먼테이션) → 스프라이트로 화면에 띄우기. 외부 의존 없이 자체 구현 가능하나 완성도가 연출력에 좌우됨.

## 3등급 비고

공통적으로 **신규 하드웨어**(버튼, 자전거, 총기형 컨트롤러, 고속 카메라) 또는 **외부 서비스/별도 스택**(VR 디바이스, 이미지 인식 API)이 필요해 현재 코드베이스 재사용률이 낮다. 행사 일정이 급하면 후순위.
