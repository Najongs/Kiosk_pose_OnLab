# 행사 운영 체크리스트

> 근거: [content/game-references.md](../content/game-references.md) —
> 허니팟 효과(A1~A5)·참여 퍼널(B1~B2)·원거리 어트랙트(C1~C3).
> **핵심 한 줄: 플레이하는 사람이 최고의 광고판이고, 아무도 안 하면 그 효과는 시작조차 안 된다.**

## 설치·배치 (개장 전)

- [ ] **통행 동선 바로 옆**에 설치 — 지나가는 사람의 시야에 화면과 플레이어가 함께 들어와야 함 (B2)
- [ ] 화면 앞 **플레이 공간 + 뒤쪽 구경 공간**을 분리 확보 — "다가가 멈춰 서서 하는 모습"이 멀리서 보일 때 유인 효과가 최대 약 6배 (A5)
- [ ] 화면은 최대한 **크게, 높게** — 구경꾼이 멀리서 점수·게이지를 읽을 수 있어야 게임을 학습하고 다가온다 (C2)
- [ ] 카메라 시야 확인: 플레이 위치에서 전신(머리~발목)이 잡히는지, 역광 없는지
- [ ] 스피커 볼륨: 주변 소음 속에서 효과음이 들리되 부담스럽지 않게
- [ ] 관리자(⚙, PIN 기본 4000)에서 카메라 재탐색 + 각 게임 1회씩 스모크 테스트

## 운영 중 (허니팟 시드)

- [ ] **개장 직후와 한산할 때 스태프가 직접 시연** — 누군가 하고 있어야 다음 사람이 온다 (A2). 특히 점프·반응속도처럼 동작이 큰 게임으로 (C3)
- [ ] 어트랙트 라이브 미러가 켜져 있는지 수시 확인 (유휴 45초 후 자동, 관리자 설정 `attractSeconds`)
- [ ] 줄이 생기면: 대기자에게 화면이 보이게 줄 방향 조정 — 구경이 곧 학습이자 홍보
- [ ] 참여를 망설이는 사람에게는 "지나가기만 해도 화면에 비쳐요" — **첫 움직임만 하게 만들면 95%는 끝까지 한다** (B1)
- [ ] 리더보드 화면(홈)을 자주 노출 — 기록 경쟁 유도

## 게임별 안내 멘트

| 게임 | 안내 |
|---|---|
| 반응속도 | "신호 뜨면 손만 번쩍! 10초면 끝나요" (참여 장벽 최소 — 첫 게임으로 추천) |
| 높이뛰기 | "잠깐 가만히 서 있다가, 힘껏 점프!" (캘리브레이션 2초 안내) |
| 팔굽혀펴기 | "옆모습이 보이게 서면 더 정확해요" |
| 스트레칭 | "화면 왼쪽 캐릭터를 따라 하면 돼요" |
| 2인 대결 | 일행이 있는 참가자에게 우선 권유 — 구경꾼 유발 최대 |

## 철수 전

- [ ] 리더보드 백업: `_internal/data/leaderboard.json` 복사 (참여 통계 겸용)
- [ ] 필요 시 리더보드 초기화(관리자 메뉴) — 다음 행사 준비
- [ ] 오류 발생 시 `_internal/error.log` 회수

## 관찰 지표 (다음 개선의 근거)

행사 중 틈틈이 기록해 두면 [engagement-improvements.md](../content/engagement-improvements.md)의
가설(리더보드 리셋·칭호 등) 검증에 쓸 수 있다:
- 어트랙트 화면 앞에서 멈추는 사람 / 실제 시작하는 사람 비율
- 게임별 선택 비율, 재도전 비율
- 어느 시간대·어떤 상황(시연 직후?)에 줄이 생겼는지
