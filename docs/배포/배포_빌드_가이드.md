# 배포용 실행파일(exe) 빌드 가이드

키오스크에 파이썬 설치 없이 배포할 수 있도록 PyInstaller 로 묶는다.
**Windows exe 는 Windows 에서 빌드해야 한다** (교차 빌드 불가).

## 1. 빌드 (개발 PC, 1회)

**⚠️ 가장 중요: 앱 의존성과 pyinstaller 가 *같은* 파이썬 환경에 있어야 한다.**
pyinstaller 가 다른 파이썬(예: 시스템 파이썬)에 설치돼 있으면 mediapipe 없는
환경 기준으로 exe 를 만들어 "빌드는 성공하는데 실행하면 ModuleNotFoundError"
가 된다. 빌드 스크립트가 사전 점검으로 이를 잡아주지만, 처음부터 한 환경에서:

```powershell
# 앱을 실행하던 그 환경(conda activate OnLab 등)에서
python -m pip install -r requirements.txt pyinstaller
python tools/build_exe.py           # → dist/OnLab/OnLab.exe
```
문제 진단이 필요하면 콘솔 로그가 보이는 버전으로:
```powershell
python tools/build_exe.py --console
```
빌드 스크립트는 시작 시 필수 모듈 임포트를 점검하고(누락 시 어떤 모듈인지
출력 후 중단), PATH 의 pyinstaller 가 아니라 **스크립트를 실행한 파이썬의**
PyInstaller(`python -m PyInstaller`)를 사용한다.

## 2. 배포
- `dist/OnLab` 폴더를 **통째로** 압축해 키오스크에 복사 → `OnLab.exe` 실행.
- 폴더(onedir) 배포인 이유: 앱이 실행 중에 설정·리더보드·카메라 캐시를
  `_internal/config/` 에 기록한다. 단일 exe(onefile)는 임시폴더에 풀려
  기록이 날아가므로 쓰지 않는다.
- 용량 약 700MB± (PySide6 + mediapipe + OpenCV 포함) — 정상.

## 3. 키오스크 설정 팁
- 부팅 시 자동 실행: `shell:startup` 폴더에 OnLab.exe 바로가기.
- 전체화면 기본. 관리자 PIN(기본 4000)으로만 설정 진입.
- BGM: `_internal/assets/bgm/` 에 음악 파일 추가.
- 자세/캐릭터 갱신은 개발 PC 에서 임포트·베이크 후 폴더 재배포
  (또는 `_internal/config`, `_internal/assets` 만 교체).

## 4. 자주 나는 문제
| 증상 | 조치 |
|---|---|
| **`No module named 'xxx'`** (트레이스백에 `_internal\...` 경로) | 빌드한 파이썬 환경에 그 모듈이 없었던 것. 위 1번대로 **한 환경에서** `python -m pip install -r requirements.txt pyinstaller` 후 재빌드. 새 스크립트는 사전 점검에서 미리 알려준다 |
| 실행 직후 종료 | `--console` 빌드로 다시 만들어 오류 확인, `_internal/error.log` 확인 |
| mediapipe 모듈 오류 | pyinstaller 최신화(`pip install -U pyinstaller`) 후 재빌드 |
| 소리 안 남 | Windows 미디어 재생 구성요소(N 에디션은 Media Feature Pack) 설치 |
| 카메라 안 열림 | 다른 앱의 카메라 점유 해제, 관리자 → 카메라 재탐색 |
| (리눅스 빌드 한정) `pyexpat ... undefined symbol` | conda 파이썬 + PyInstaller 조합의 알려진 문제 — 번들 `_internal/libexpat.so.1` 을 conda env 의 것으로 교체. Windows 와 무관 |

> 코드가 바뀌면 exe 도 다시 빌드해야 한다 — git pull 만으로는 exe 에 반영되지
> 않는다 (§5). 최근 커밋에는 새 모듈(core/games, ui/game_* 등)이 많이 추가됐다.

## 5. 업데이트 배포
git 으로 코드 갱신 후 재빌드 → `dist/OnLab` 교체.
설정 유지가 필요하면 기존 `_internal/config/app_config.json`,
`leaderboard.json`, `refs*.json`, `camera_cache.json` 을 백업 후 복원.
