## 1. 개요

- **1.1 목적**: 〔 본 시스템이 제공하는 가치·목표를 1~2문단으로 〕
- **1.2 범위**: 〔 포함/제외 범위 (콘텐츠 4종, 관리자, 리더보드 등 / 제외 항목) 〕
- **1.3 용어·약어**: 키포인트, 스켈레톤, ROM(가동범위), FPS, TTS 등 〔 〕
- **1.4 참고 문서**: 설계서, 회의록, 벤더 스펙 〔 〕

---

## 2. 시스템 개요

**2.1 전체 구성 (파이프라인)**

```
Camera → Frame Acquisition → Pose Estimation → Skeleton
      → Gesture/Pose Recognition → Interaction Engine → Application → Display
                                                              ↑ Admin / Leaderboard / Data
```

**2.2 이해관계자 / 사용자**

| 구분 | 설명 |
|------|------|
| 일반 사용자 | 〔 연령·목적 〕 (예) 전시·행사 방문객 |
| 관리자 | 자세 세트·임계값·리더보드 관리 |
| 운영자 | 설치·전원·일상 운영 |

**2.3 운영 환경**: 〔 설치 장소, 조도, 이용자와 카메라 거리, 소음 〕

## 3. 기능 요구사항 (FR)

> ID 규칙: FR-<영역>-<번호>, 검증: 시연/테스트/측정

**3.1 공통 인터랙션 엔진**

| ID | 요구사항 | 우선순위 | 검증 |
|----|----------|:---:|------|
| FR-ENG-01 | 카메라 프레임에서 스켈레톤(키포인트) 실시간 추출 | M | 시연 |
| FR-ENG-02 | 군중 속 주 대상 1명 선택·추적 (겹침 대응) | M | 시연 |
| FR-ENG-03 | 다중 인원(2인) 좌/우 배정 및 동시 처리 | S | 시연 |
| FR-ENG-04 | 상태 흐름: 대기→감지→카운트다운→수행→결과 | M | 시연 |
| FR-ENG-05 | 저신뢰 관절(가림) 자동 가중치 축소/제외 | S | 테스트 |

## 6. AI / 모델 요구사항

- **6.1 백본**: 〔 MediaPipe Pose(권장) / OpenPose 〕, 키포인트 수 〔 33 / 25 〕
- **6.2 채점 로직**: 자세별 관절 각도 스펙(목표·허용오차·가중치), 좌우 대칭·가동범위(ROM), 유지시간
- **6.3 평가 지표·수용 기준**: 4.의 NFR-A/P와 연계, 데이터셋 출처 〔 자체 촬영/공개 〕, 라벨 주체 〔 〕
- **6.4 대상 하드웨어 성능 목표**: 〔 Jetson에서 FPS·지연 〕
